## 1. Concept & Vision

**一句话**：AI玩家与人类玩家同台博弈的狼人杀网页游戏。

**核心理念**：让玩家体验与真人无异的AI狼人杀对战——AI会骗人、会分析、会倒钩、更会焊跳。融合社交推理的刺激与AI博弈的新奇感。

**差异化**：
- 市面仅有的"AI+人类"混合狼人杀
- 参考图片风格：深紫星空 + 圆形玩家卡片 + 实时日志面板
- 支持12人标准局

## 2. Design Language

### 视觉风格
- **主题**：深紫星空 + 霓虹光效（赛博朋克狼人杀）
- **背景**：深紫色(#1a0a2e) + 星点粒子动画
- **氛围**：神秘、紧张、科幻感

### 配色方案
```
Primary:      #6c5ce7 (紫色 - 魔法/夜晚)
Secondary:    #00cec9 (青色 - 好人阵营)
```

- 主标题：`"Orbitron", sans-serif`（科幻感）

### 动效
- 夜晚转换：渐暗 + 月亮动画
- 死亡：卡牌翻转 + 灰化
- 投票：数字跳动
- 卡片悬停：微微浮起 + 光晕

## 4. Game Rules

### 角色配置（12人标准局）
| 阵营 | 角色 | 数量 | 技能 |
|------|------|------|------|
| 好人 | 村民 | 4 | 无特殊技能，纯分析 |
| 好人 | 预言家 | 1 | 每晚查验1人 |
| 好人 | 女巫 | 1 | 解药+毒药各1 |
| 好人 | 守卫 | 1 | 每晚守护1人 |
| 好人 | 猎人 | 1 | 死亡时可带走1人 |
| 狼人 | 普通狼人 | 3 | 每晚刀人 |
| 狼人 | 狼王 | 1 | 被票/毒死可带走1人 |

### 胜负条件
- **好人胜利**：所有狼人出局
- **狼人胜利**：狼人数量 >= 好人数量

### 游戏流程

#### 夜晚阶段
1. **天黑请闭眼** → 所有人卡片暗化
2. **守卫行动** → 选择守护对象
3. **狼人行动** → 狼人讨论（3轮）→ 投票刀人
4. **女巫行动** → 救人/毒人（首夜有系统提示）
5. **预言家行动** → 查验1人
6. **结算** → 宣布死亡（若有）
7. **天亮** → 存活玩家发言

#### 白天阶段
1. **死亡公告** → 宣布昨夜死亡者
2. **遗言** → 死亡玩家最后发言
3. **发言环节** → 存活玩家轮流发言
4. **投票环节** → 票选嫌疑最大者
5. **处决** → 票数最多者出局
6. **判断胜负** → 检查胜利条件
7. **进入黑夜** → 下一轮

## 5. AI Player System

### AI架构
```
AI Player
├── LLM Adapter (OpenAI / Anthropic / DeepSeek)
├── Role Prompt (角色设定)
├── Game State Context (当前局势)
└── Response Parser (输出解析)
```

### AI角色提示词模板
```markdown
你是狼人杀游戏中的一名玩家。
你的角色是：【角色名】
阵营：【好人/狼人】
当前游戏轮次：第N天
存活玩家：X人

你的任务：
- 好人：分析局势，找出狼人，带动投票
- 狼人：隐藏身份，配合队友，误导好人

发言规则：
- 每轮发言不超过50字
- 可以分析其他玩家的行为
- 可以质疑或保某位玩家
```

### AI行为模式
1. **村民**：分析投票记录，尝试找出狼人
2. **预言家**：适时报查验结果（可选择焊跳）
3. **女巫**：谨慎用药，适时报信息
4. **狼人**：配合焊跳，扰乱好人视线

## 6. Technical Architecture

### 技术栈
- **前端**：Vanilla HTML/CSS/JavaScript（单文件，利于快速迭代）
- **后端**：Node.js + Express（游戏逻辑）
- **实时通信**：WebSocket（Socket.IO）
- **AI接口**：OpenAI Compatible API

### 项目结构
```
werewolf-game/
├── index.html          # 游戏主页面
├── css/
│   └── style.css       # 样式
├── js/
│   ├── game.js         # 游戏主逻辑
│   ├── ai.js           # AI玩家
│   ├── ui.js           # 界面渲染
│   └── network.js      # WebSocket通信
├── server/
│   └── index.js        # Node.js服务器
├── prompts/
│   └── role_prompts.js  # AI角色提示词
└── assets/
    └── sounds/         # 音效
```

### API设计
```javascript
// WebSocket事件
- game:start        // 游戏开始
- game:phase        // 阶段切换
- player:action     // 玩家行动
- player:speak      // 玩家发言
- player:vote       // 投票
- player:die        // 玩家死亡
- game:end          // 游戏结束

// HTTP API
POST /api/ai/chat  // AI对话
POST /api/game/new // 创建游戏
GET  /api/game/:id // 获取游戏状态
```

## 9. Edge Cases & Error Handling

| 场景 | 处理方式 |
|------|----------|
| AI API超时 | 使用默认回复，提示"AI思考中..." |
| 狼人全掉线 | 好人自动胜利 |
| 玩家断线 | 保留座位3分钟，超时判离线 |
| 平票 | 重新投票，最多3次 |
| AI胡言乱语 | 过滤敏感词，限制长度 |
